# 我的分红工具

GitHub Pages：https://datiancailty.github.io/stock-dashboard/

## 功能

- Part 1：用户名与密码登录后，可用股票代码、中文、拼音或拼音首字母模糊搜索，并把股票永久加入 GitHub 云端自选股。
- Part 1 每只股票都可点击“编辑持仓/成本”，修改持仓股数和每股成本价，并立即重新计算市值与预计分红；这两项个人数据保存在当前浏览器。
- Part 2：正式股息率及 5%、5.5%、6%、6.5%、7% 价格网格；AI公告预估独立展示，不覆盖正式值。
- Part 3：登记日、除息日和派息日。
- Part 4：分红公告/新闻增量记忆、来源链接、状态、可信度和可审计计算公式。
- 手动覆盖年报分红、中报分红和当前股价仍保留，只存在当前浏览器。

## 正式分红口径

1. 选取上一完整财务年度。
2. 使用该年度已实施的税前现金分红。
3. 年报与中报同时存在时相加。
4. 过滤“不分配不转增”。
5. `正式股息率 = 正式年度每股分红 / 当前股价 × 100%`。
6. `目标股价 = 正式年度每股分红 / 目标股息率`。

## 公告记忆与AI预估

- 每天北京时间 08:30 使用东方财富妙想资讯搜索合并检查一次全部自选股，消耗 1 次调用。
- 从 `live/news-memory.json` 读取历史记忆，根据股票、标题、日期和来源生成唯一指纹。
- 已处理内容不会再次加入或重新计算；只保存新增公告/新闻。
- 优先识别公告直接披露的每股分红，其次才按“现金分红总额÷总股本”或“归母净利润×分红比例÷总股本”推算。
- 每条记录保存原始来源、摘要、输入数字、公式、状态和可信度。
- “已实施”“正式预案”“政策预估”分开显示。政策预估不会覆盖正式股息率。
- 登录用户可点“立即检查新公告”，通过提交刷新请求触发 GitHub Actions。

## 行情更新与额度

A股数据通过东方财富妙想 API 获取，`MX_APIKEY` 只保存在 GitHub Actions Secret。

工作日北京时间：

- 09:30—11:30：60次，约每120秒一次
- 13:00—15:30：70次，约每129秒一次
- 行情计划共130次
- 公告每日1次，正常工作日总计131次，仍留19次机动额度
- 新增股票时，行情额外触发1次；公告最多额外触发2次（一次增量扫描，加一次新股票历史补抓）

实时 `market.json` 与 `news-memory.json` 写入 `live` 分支，网页打开时只读取已保存结果，不会把妙想密钥暴露给浏览器，也不会因每次打开重复消耗额度。

## 登录安全模型

GitHub Pages 没有服务端。网页使用 PBKDF2-SHA256（60万次）和 AES-256-GCM 保存**加密后的**仓库限定凭据；登录密码不保存，只有正确密码才能在当前浏览器内存中解密。GitHub fine-grained token 必须：

- 仅允许 `datiancailty/stock-dashboard` 一个仓库
- Repository permissions → Contents：Read and write
- Metadata：Read-only
- 不授予其他仓库或账户权限

这比把 token 明文放在网页安全，但公开静态站仍允许离线尝试密码，因此必须使用随机强密码。退出或刷新页面后，解密凭据从内存清除。
